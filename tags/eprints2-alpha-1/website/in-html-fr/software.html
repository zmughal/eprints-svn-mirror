Caractéristiques

Facilité d'installation et de lancement. Dans le cas le plus simple, éditez les fichiers de configuration, lancez le programme d'installation et le site fonctionne. Possibilité d'enregistrer les documents au format que vous (administrateur des archives) préférez. Tout article de recherche (ou e-print) peut être enregistré sous plusieurs formats. L'administrateur des archives peut également choisir les métadonnées pour chaque e-print. Ceci est possible en trois ou quatre étapes:

  1. Choisir un nombre maximum d'éléments de métadonnées pouvant être enregistrés (par exemple, "auteurs", "titre", "revue", "n° de volume", etc.)
  2. Choisir les types d'e-print qui seront enregistrés (par exemple, article de revue à comité de lecture, thèse, rapport technique, pre-print non publié)
  3. Choisir les éléments de métadonnées qui devront être enregistrés pour tous les e-prints de même type, en précisant les éléments obligatoires.
  4. Choisir comment les éléments de métadonnées doivent être reprises dans les Open Archives. (Si nécessaire, l'interopérabilité peut être inactivée, mais c'est fortement déconseillé.)
 Les e-prints peuvent être archivés selon un plan de classement hiérarchisé. Ce classement peut être utilisé pour visualiser et effectuer une recherche dans les archives. La déposition de tous les e-prints s'effectue via une interface web simple mais extrêmement puissante. Les articles peuvent être téléchargés vers le serveur comme de simples fichiers, dans un format compressé (un fichier .zip par exemple), ou automatiquement et intégralement copiés d'un site ou ils sont déja stockés, en indiquant l'URL. L'intégrité des données est vérifiée automatiquement sans que l'administrateur ait besoin d'intervenir. Des bouts de routines permettent à des sites individuels d'intégrer leur propre routine de vérification d'intégrité si besoin est. Ces vérifications simplifient considérablement le travail de l'administrateur du site. Des métadonnées peuvent être associées aux auteurs. L'administrateur peut là encore choisir les éléments de métadonnées qui seront enregistrés avec les coordonnées de chaque auteur. Les utilisateurs peuvent s'abonner en tant qu'auteur ou lecteur, grâce à un formulaire web ou un compte e-mail géré automatiquement. Les utilisateurs peuvent s'abonner à un service d'e-mails. Chaque utilisateur précise ses domaines favoris et peut alors choisir de recevoir par e-mail les résumés des nouvelles dépositions dans ces domaines. Les articles déposés (si l'administrateur le souhaite) font l'objet d'une procédure de modération. Les papiers déposés sont mis en attente jusqu'à être approuvés par le modérateur, rejetés catégoriquement ou retourné à l'auteur pour modifications. (Cette modération est spécifique à chaque organisme participant aux archives : à CogPrints, nous l'utilisons pour nous assurer que le sujet traité est approprié et que toutes les figures apparaissent correctement.) Cette procédure de modération s'effectue également grâce à une interface web. En fait, quand le site fonctionnera, il y aura très peu de travaux de maintenance qui ne pourront pas être réalisés par l'interface web de maintenance.

Configuration requise

L'un des objectifs majeurs de ce travail a été de rendre ce système fonctionnel à moindre coût. Les seuls frais seront le coût du matériel. Le matériel et les logiciels requis sont:

Un PC fonctionnant sous UNIX. Evidemment, plus le PC est rapide, mieux ça fonctionne, mais tout processeur Intel Pentium II permettra de bonnes performances. Un système d'exploitation UNIX. Linux (un système UNIX très performant et gratuit) suffit amplement. En fait, il s'agit de la plate-forme de développement. Le serveur WWW Apache, un autre logiciel gratuit de qualité professionnelle, souvent inclus dans les distributions Linux, tel que RedHat. Le langage de programmation Perl, également compris avec la plupart des distributions Linux. (Ainsi qu'un petit nombre de modules supplémentaires, plus de détails seront disponibles ultérieurement.) Le module mod_perl pour Apache, qui augmente de manière significative la performance des scripts Perl. Notez que le mod_perl fourni avec RedHat 6.2 (architecture i386) ne fonctionne pas, il doit être remplacé par ce RPM. La base de données MySQL, qui est un système de base de données gratuit à usage non commercial. (Vous trouverez le texte de la licence ici.) L'accès à la base de données s'effectue grâce à un seul module, et le gestionaire de la base de données pourra facilement être remplacé si vous le souhaitez. Et le logiciel eprints.org!

Quand les logiciels sont installés, la mise en place du logiciel eprints.org se fait en éditant quelques fichiers de configuration et en lançant le programme d'installation. Les archives fonctionnent, vous pourrez dès lors utiliser le système à votre guise (ou à celle de votre organisme de travail).
